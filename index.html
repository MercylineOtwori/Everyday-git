Everyday Git

1. Get Started

Set up your development environment by creating a folder named everyday-git. Inside it, create an index.html file and an images folder for storing screenshots.

2. What is Git?

Git is a version control system used to track changes in source code. Version control systems allow developers to collaborate and manage code history. Other examples include Subversion, Mercurial, and CVS. In this blog, weâ€™ll focus on Git and GitHub.

3. Installing Git

Check if Git is installed:

git --version

If not installed, download Git from git-scm.com and follow the installation instructions for your OS.

4. Configuring Git

git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

5. Initializing a Git Repository

Initialize a Git repository in your project directory:

git init

6. Adding Files to the Staging Area

The staging area holds changes before a commit. Tracked files are monitored by Git, while untracked files are not.

git add index.html
git add .

7. Committing Changes

git commit -m "Initial commit"

Good commit messages are short and descriptive.

git log
git checkout <commit_hash>

8. Branches

Branches allow you to work on different features without affecting the main codebase. Theyâ€™re important for collaboration and feature testing.

9. Creating a Branch

git branch new-feature

10. Viewing All Branches

git branch

11. Switching Active Branches

git checkout new-feature

12. Merging a Branch

git checkout main
git merge new-feature

13. Deleting a Branch

git branch -d new-feature

14. Next Steps

Next, learn how to use GitHub for remote repositories, and try using commands like git push, git pull, git clone. Explore GUIs like GitHub Desktop or use Git with Visual Studio Code.

15. Deploy Your Page

Deploy using GitHub Pages or Vercel:

  • GitHub Pages: Push to GitHub, go to Repo Settings > Pages > Deploy from main branch
  • Vercel: Connect your GitHub repo and deploy directly
